# Reporose Project Context

## Overview
Reporose is a repository scanner that builds a structural map of a codebase, including functions, imports, exports, and dependencies. It generates a `map.json` file that can be visualized in 3D or used for AI-assisted code navigation. Matter of Fact,,, A map for this repo can also be generated by running npm run analyze on the root of this project

## Architecture

### Core Components
- **Scanner** (`src/core/scanner.js`): Parses JavaScript/TypeScript files using Babel AST parser. Extracts functions, classes, imports, exports, and builds file metadata.
- **Mapper** (`src/core/mapper.js`): Computes dependency links between files, detects circular dependencies, and identifies dependency networks.

### AI Providers (`src/ai/`)
- **Ollama** (`ollama.js`): Local AI via Ollama API. Default model: `qwen2.5-coder:7b`
- **Ollama Cloud** (`ollama.js`): Cloud version of Ollama with API key authentication. Supports curated models like `gpt-oss:20b-cloud`
- **Anthropic** (`anthropic.js`): Anthropic Claude API via Messages API. Default model: `claude-haiku-4-5`
- **Groq** (via OpenAI): Fast inference via Groq API using OpenAI-compatible endpoint
- **OpenAI** (`openai.js`): OpenAI API compatibility
- **OpenRouter** (`openrouter.js`): Cloud AI with free tier
- **Local** (`local.js`): Local GGUF models via node-llama-cpp with tier-based presets (low/medium/high)
- **Heuristic** (`heuristic.js`): Rule-based, no AI - generates descriptions from AST patterns
- **None**: Disables AI description generation entirely
- **Summarizer** (`summarizer.js`): Orchestrates file/function description generation
- **Config** (`config.js`): Manages AI provider configuration and presets

### CLI & Server
- **CLI** (`src/cli/commands.js`, `bin/cli.js`): Commands: `init`, `analyze`, `map`, `summarize`, `serve`, `config`, `preset`
- **Server** (`src/server/server.js`): Express server serving 3D visualization

## Key Configuration
- Config stored at `<repo>/.reporose/config.json` or `~/.reporose/config.json`
- Cache stored at `<repo>/.reporose/cache/summaries.json`
- Output: `<repo>/.reporose/index.json` (lightweight index) + `files/*.json` (full file data)

## Default AI Settings

### Ollama Cloud
- Model: `gpt-oss:20b-cloud`
- Context: 32000 tokens
- Temperature: 0
- Max predict: 800 tokens
- API key required via `OLLAMA_API_KEY` environment variable

### Ollama (Local)
- Model: `qwen2.5-coder:7b`
- Context: 32000 tokens
- Temperature: 0
- Max predict: 10000 tokens
- Keep-alive: Model stays loaded during entire scan, unloaded only at end

### Anthropic
- Model: `claude-haiku-4-5`
- API key required via `ANTHROPIC_API_KEY` environment variable
- Streaming support via SSE

### Local Provider Tiers
- **Low**: ctx 2048, gpu_layers 0, file_tokens 180, fn_tokens 90 (1B-3B GGUF)
- **Medium**: ctx 4096, gpu_layers 24, file_tokens 280, fn_tokens 110 (7B GGUF)
- **High**: ctx 8192, gpu_layers 99, file_tokens 480, fn_tokens 140 (13B+ GGUF)

## File Processing Flow
1. **Phase 1 (Scan)**: Walk repo, parse AST, extract functions/imports/exports
2. **Phase 2 (Map)**: Compute dependency graph, detect cycles/networks
3. **Phase 3 (Summarize)**: AI generates file descriptions with searchable tags
4. **Phase 4 (Visualize)**: 3D visualization via web interface

## New in v0.2.0 (Split Storage)
- **Split Map Storage**: Large repos now use `index.json` + `files/*.json` instead of a single large `map.json`
- **Searchable Tags**: AI generates tags for each file to improve searchability
- **AI Pre-filter**: Before summarizing, AI reviews the project structure to skip auto-generated files
- **Next.js Support**: Detects implicit connections between routes and layouts

## Important Notes
- Function descriptions are intentionally left empty to reduce token usage
- Only file descriptions are generated by AI
- Model is kept "hot" (loaded) throughout the entire summarization phase
- **Backward Compatible**: Old `map.json` format is still readable; run `reporose analyze` to upgrade to new format

# RepoRose Brand & UI Guidelines

## Brand Identity
- **Name:** RepoRose
- **Font:** 'Red Rose', serif (Google Fonts)
- **Font URL:** `https://fonts.googleapis.com/css2?family=Red+Rose:wght@300;400;600;700&display=swap`

## Color Palette (5-Color System)
| Token | Hex | Usage |
|-------|-----|-------|
| `white` | `#FFEDFA` | Canvas White - main backgrounds, replacement for white |
| `primaryLight` | `#FFB8E0` | Rose Tint - soft highlights, secondary backgrounds, subtle accents |
| `primary` | `#EC7FA9` | Vivid Rose - primary brand color, buttons, active states |
| `primaryDark` | `#BE5985` | Deep Magenta - borders, secondary text, hover states |
| `black` | `#1A050F` | Night Rose - primary text, contrast, replacement for black |

## Typography Scale
- **H1:** Bold, 4xl (36px)
- **H2:** Semi-Bold, 2xl (24px)
- **Body:** Regular, lg (18px), leading-relaxed
- **Font Weights:** 300 (light), 400 (regular), 600 (semi-bold), 700 (bold)

## UI Patterns

### Cards/Containers
- Border radius: `rounded-2xl` (16px), `rounded-3xl` (24px)
- Shadows: `shadow-sm` default, `hover:shadow-md` on interaction
- Border: subtle with 20-30% opacity of `primaryDark`
- Backdrop blur: `backdrop-blur-sm` for glassmorphism

### Buttons/Interactive
- Primary: `primary` background, `black` text
- Hover states use `primaryDark`
- Copy buttons: transition between `white` (10% opacity) and `primary`

### Code Blocks
- Background: `black` (`#1A050F`)
- Text: `primaryLight` (`#FFB8E0`)
- Terminal dots: `primaryLight`, `primary`, `primaryDark`
- Border: `primaryDark` with 40% opacity

### Section Headers
- Left border accent: 4px solid `primary`
- Format: `01. Title` numbering style
- Description text below with opacity-80

### Logo Usage
- Aspect ratio: 1:1
- Sizes used: 48px (small), 64px (default), 80px (header)
- Always maintain sufficient padding around logo

## Selection Style
```css
selection:bg-[#EC7FA9] selection:text-[#1A050F]
```

## Spacing Scale
- Page padding: `p-6 md:p-12 lg:p-20`
- Section gaps: `space-y-20` (80px), `space-y-8` (32px)
- Grid gaps: `gap-4 md:gap-6`
- Card padding: `p-4` (16px), `p-6` (24px), `p-8` (32px)

## Animation Defaults
- Transition duration: `duration-300`
- Hover scale: `group-hover:scale-105`
- Transform origin: `origin-bottom` for cards

## Component Inventory
- `RoseLogo` - Abstract rose SVG with gradient petals
- `ColorSwatch` - Color palette display card
- `CodeSnippet` - Copyable terminal-style code block
